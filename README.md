# IT Helpdesk Incident Management Analytics Dashboard (Power BI)

## 📌 Project Overview

This project presents an interactive **IT Helpdesk Incident Management Analytics Dashboard** built using Power BI. The dashboard analyzes incident lifecycle data to uncover operational trends, process inefficiencies, and resolver performance insights.

With my professional background in **Incident Management**, I developed this project to transition into **Data Analytics** by applying domain knowledge to real-world incident data analysis.

The dashboard simulates a real IT Service Management (ITSM) environment, similar to ServiceNow or Jira.

---

## 🎯 Project Objectives

The key objectives of this project are:

- Monitor incident volume and backlog trends  
- Analyze Mean Time to Resolution (MTTR) performance  
- Identify process inefficiencies such as reopens and multiple handoffs  
- Evaluate resolver workload distribution and efficiency  
- Provide actionable insights to improve incident resolution performance  

---

## 📊 Dashboard Structure

The dashboard consists of three analytical sections:

### 1️⃣ Incident Overview & Trends

Provides high-level operational insights:

- Total incidents and open backlog monitoring  
- Incident reopen rate analysis  
- Incident creation vs resolution trends  
- MTTR trend analysis over time  
- Incident distribution by priority and status  

**Purpose:**  
Helps monitor overall operational health and incident resolution efficiency.

---

### 2️⃣ Process Quality & Root Cause Insights

Focuses on identifying resolution delays and process inefficiencies:

- Impact of incident reopens on MTTR  
- Impact of multiple resolver handoffs on resolution time  
- Reopen rate by incident priority  
- Identification of longest resolution incidents  
- Root cause analysis of process bottlenecks  

**Purpose:**  
Helps identify factors affecting resolution efficiency and process quality.

---

### 3️⃣ Resolver Performance & Workload Intelligence

Analyzes resolver workload and efficiency:

- Resolver workload distribution  
- Average handling time per resolver  
- Resolver reassignment rate analysis  
- Workload imbalance identification  
- Resolver performance comparison  

**Purpose:**  
Helps optimize workload distribution and improve team performance.

---

## 📈 Key Metrics Used

- Total Incidents  
- Open Incidents  
- Reopen Rate (%)  
- Mean Time to Resolution (MTTR)  
- Resolver Assignments  
- Reassignment Rate (%)  
- Average Assignments per Resolver

## 🔐 Data Privacy and Masking

To comply with data privacy and security standards, sensitive fields have been masked and anonymized.

Masked fields include:

- Application / Project  
- Resolver ID  
- Reported By  

This ensures no personally identifiable or sensitive information is exposed.

---

## 💡 Key Insights

The dashboard reveals several operational insights:

- Incidents with multiple resolver handoffs have significantly higher resolution time  
- Reopened incidents consistently show increased MTTR  
- Resolver workload distribution is uneven across teams  
- Resolution trends provide visibility into operational efficiency  

These insights help improve incident management efficiency and operational decision-making.
